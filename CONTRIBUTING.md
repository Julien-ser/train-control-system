# Contributing to Train Control System

Thank you for your interest! This document provides guidelines for contributing.

## Development Workflow

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/train-control-system.git`
3. Create a branch: `git checkout -b feature/your-feature`
4. Make changes and add tests
5. Ensure all tests pass: `pytest tests/`
6. Commit: `git commit -m "Add: descriptive message"`
7. Push: `git push origin feature/your-feature`
8. Open a Pull Request

## Code Style

- Follow PEP 8
- Use type hints where appropriate
- Write docstrings for modules, classes, and functions
- Keep functions small and focused
- Write tests for new functionality

## Testing

Run `pytest tests/` before submitting a PR. Ensure all tests pass.

## Reporting Issues

- Use GitHub Issues
- Describe the problem, steps to reproduce, expected behavior
- Include system details (Python version, OS)

## Questions?

Open an issue for discussion.
